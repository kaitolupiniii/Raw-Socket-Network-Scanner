# Raw-Socket Network Scanner

A Rust-based network scanner that constructs and sends raw Ethernet/IP/UDP/TCP packets by manually building L2/L3/L4 headers. This educational tool demonstrates low-level packet construction and network protocol implementation.

## Build Instructions

### Prerequisites
- Rust toolchain (1.70.0 or later)
- Cargo package manager
- Linux environment (recommended)

### Building the Project

```bash
# Clone or download the project
cd Raw-Socket-Network-Scanner

# Build in release mode (recommended)
cargo build --release

# Build in debug mode
cargo build

# Run tests
cargo test
```

The binary will be available at `target/release/raw_scanner`.

## Usage

### Command-Line Interface

The scanner supports the following command-line flags:

| Flag | Description | Example |
|------|-------------|---------|
| `--src_ip` | Source IPv4 address for IP header | `--src_ip=192.168.1.100` |
| `--dst_ip` | Destination IPv4 address for IP header | `--dst_ip=8.8.8.8` |
| `--dest_port` | Destination port for L4 protocol | `--dest_port=80` |
| `--src_mac` | Source MAC address for Ethernet header | `--src_mac=aa:bb:cc:dd:ee:ff` |
| `--dst_mac` | Destination MAC address for Ethernet header | `--dst_mac=11:22:33:44:55:66` |
| `--l4_protocol` | Layer-4 protocol (udp/tcp) | `--l4_protocol=tcp` |
| `--timeout_ms` | Timeout between operations in milliseconds | `--timeout_ms=2000` |
| `--debug_file` | Write debug output to file | `--debug_file=scan.pcap` |
| `--debug_format` | Debug file format (json/pcap) | `--debug_format=pcap` |
| `--ip_bitfield` | IP header flags/bitfield value | `--ip_bitfield=0x04` |
| `--dry_run` | Construct packets without sending | `--dry_run` |

### Run Examples

#### Basic Scanning
```bash
# UDP scan to localhost
target/release/raw_scanner --dst_ip=127.0.0.1 --dest_port=8080

# TCP scan to remote host
target/release/raw_scanner --l4_protocol=tcp --dst_ip=192.168.1.1 --dest_port=80

# Custom source IP in packet header
target/release/raw_scanner --src_ip=10.0.0.1 --dst_ip=192.168.1.100 --dest_port=443
```

#### Debug Output Examples
```bash
# Save packet construction to PCAP file (Wireshark compatible)
target/release/raw_scanner --dst_ip=8.8.8.8 --dest_port=53 --debug_file=dns_scan.pcap --debug_format=pcap

# Save packet details to JSON
target/release/raw_scanner --l4_protocol=tcp --dst_ip=192.168.1.1 --debug_file=tcp_scan.json --debug_format=json

# Dry run - construct packets without sending
target/release/raw_scanner --src_ip=1.2.3.4 --dst_ip=192.168.1.1 --dry_run --debug_file=constructed.pcap --debug_format=pcap
```

#### Individual Flag Testing (as required by specification)

### Monitoring Results

To see the network traffic generated by the scanner:

```bash
# Monitor UDP traffic
sudo tcpdump -i lo -n udp port 8080

# Monitor TCP traffic (SYN packets)
sudo tcpdump -i lo -n tcp port 8080

# Monitor all traffic on specific interface
sudo tcpdump -i eth0 -n host 192.168.1.100
```

## Required Privileges

### Current Implementation (Sudo-Less Operation)
The scanner is designed to run **without root privileges** in its current implementation:

- **Regular User Mode**: Uses standard UDP/TCP sockets that don't require elevated privileges
- **Packet Construction**: All raw packet construction happens in userspace
- **Actual Transmission**: Uses standard system calls available to all users

```bash
# No sudo required for these operations:
target/release/raw_scanner --dst_ip=127.0.0.1 --dest_port=8080
target/release/raw_scanner --l4_protocol=tcp --dst_ip=192.168.1.1 --dry_run
```

### Advanced Usage (Optional)
For educational purposes, the code includes raw socket infrastructure that would require root privileges if fully implemented:

```bash
# This would require sudo for true raw socket access:
sudo target/release/raw_scanner --dst_ip=192.168.1.1 --dest_port=80
```

### PCAP Format
- Wireshark-compatible packet capture files
- Can be analyzed with: `wireshark scan.pcap` or `tcpdump -r scan.pcap`
- Contains full Ethernet frames with constructed headers

## Technical Features

### Protocol Support
- ✅ **Ethernet (L2)**: Manual frame construction with custom MAC addresses
- ✅ **IPv4 (L3)**: Complete header with proper checksums and bitfield support
- ✅ **UDP (L4)**: Full protocol implementation with checksum verification
- ✅ **TCP (L4)**: SYN packet construction with proper header fields

### Advanced Capabilities
- **IP Bitfield Support**: Custom IP header flags including research "evil bit"
- **Checksum Calculation**: Manual computation of IP and transport layer checksums
- **Dry Run Mode**: Safe packet construction without network transmission
- **Cross-Platform**: Standard Rust implementation works on major platforms
